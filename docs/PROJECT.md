### 주차 유저란?

- **정의**: 캐릭터의 성장(레벨업, 전투력 향상)을 의도적으로 멈추고, 특정 성장 구간에 고정된 채 **주간 보스 레이드를 반복**하여 **메소(게임 내 화폐)를 대량 생산**하는 유저. 사냥보다 보스 드롭 수익이 주된 수익원임
- **문제**: 이들의 대규모 메소 공급은 인플레이션을 유발하며, 게임 경제 밸런스를 교란함
- **현재 한계**: 일반 유저와 행동 패턴이 외관상 유사해 **규칙 기반 타겟팅이 어려운 상황**

## 💡 가설 3개

### 가설 1 — 클러스터링으로 주차 유저 집단 분리 가능성

> **"주차 유저는 일반 유저와 플레이 패턴 feature 상에서 구별되는 군집을 형성할 것이다."**
> 
- **검증 방법**: K-Means 또는 DBSCAN 클러스터링 (Elbow Method / Silhouette Score로 최적 k 선정)
- **주요 Feature 후보**:
    - 지난 2년간 레벨 변화량 (Δlevel)
    - 지난 2년간 전투력 변화량 (Δ전투력)
    - 지난 2년간 유니온 레벨 변화량 (Δunion)
    - 심볼 성장 수준 (아케인/어센틱)
- **기대 결과**: 성장 지표(레벨·전투력·유니온) 변화가 거의 없는 군집이 별도로 형성됨
- **EDA 판정**: ✅ **조건부 가능** — arcane_stagnant 이진화 + delta_cp Winsorize 전처리 후 실시

---

### 가설 2 — 주차 유저는 특정 레벨 구간·직업군에 불균형하게 집중 분포한다

> **"주차 유저 군집은 전체 유저 대비 특정 레벨 구간 및 직업군에서 통계적으로 유의미하게 높은 비율로 나타날 것이다."**
> 
- **검증 방법**: 카이제곱 검정(Chi-Square Test)으로 레벨 구간별·직업별 주차 유저 비율 분포 검정
- **검정 대상**:
    - 레벨 구간별 (~260 / 260~280 / 280~300 / 300+) 주차 유저 비율
    - 직업군별 주차 유저 비율 (전사 / 마법사 / 궁수 / 도적 / 해적 5계열)
- **유의수준**: α = 0.05
- **EDA 판정**: ✅ **가능** — 레벨 4구간 Chi2=663.39 (p≈0), 직업 5계열 Chi2=24.88 (p=0.0001), 모든 셀 기대 빈도 ≥ 5
- **⚡ Action Required**: 주차 유저가 집중된 레벨 구간과 직업군을 데이터로 특정함으로써, 넥슨이 **해당 구간의 보스 드롭률·메소 수급량을 선별 조정**할 수 있는 근거 제공

---

### 가설 3 — 도출한 Rule-Based 기준은 낮은 오분류율로 주차 유저를 식별할 수 있다

> **"Feature Importance 기반으로 도출한 Rule-Based 타겟팅 기준은, 일반 유저를 주차 유저로 잘못 분류하는 비율(False Positive Rate)을 허용 가능한 수준으로 유지하면서 주차 유저를 식별할 수 있다."**
> 
- **검증 방법**:
    - Random Forest 또는 XGBoost로 Feature Importance 산출 후 핵심 feature 기반 Rule 도출
    - Rule의 분류 성능을 Precision / Recall / F1-Score / False Positive Rate로 평가
    - ROC-AUC 곡선으로 Threshold별 Trade-off 시각화
- **EDA 판정**: ✅ **가능** — 주차 프록시 비율 23.7% (클래스 불균형 없음), H1 Silhouette Score > 0.4 조건부
- **⚡ Action Required**: 디렉터가 "그분들만 타겟팅해서 줄이는 게 어려웠다"고 언급한 문제에 대해, **일반 유저 피해를 최소화하면서 적용 가능한 Rule을 수치 근거와 함께 제시**

## 🔍 EDA 주요 발견 사항 (`eda.ipynb`)

### 데이터 규모 (전처리 후)

| 항목 | 수치 |
| --- | --- |
| 원본 행 수 | 1,497행 |
| 유효 행 수 (`df_clean`) | 1,463행 (NaN 34행 제거) |
| 전체 num_valid_months | 전원 24개월 (결측 없음) |
| 주차 프록시 비율 (Δ레벨 ≤ 0.1) | **23.7%** (346명) |
| 280+ 레벨 집중도 | 75.7% (유니온 랭킹 샘플링 편향) |

### 레벨 구간 분포

| 레벨 구간 | 인원 | 주차 프록시 비율 |
| --- | --- | --- |
| ~260 | 98명 | **88.8%** (저레벨 집중) |
| 260~280 | 317명 | 61.8% |
| 280~300 | 1,048명 | 6.1% |
| 300+ | 0명 | — (레벨 max=300, pd.cut 오른쪽 경계 처리) |

> 주차 프록시는 임시 레이블 (Δ레벨 ≤ 0.1) — 클러스터링 결과와 교차 검증 필요

### 핵심 피처 통계 (`avg_monthly_delta_*`)

| 피처 | 중앙값 | 특이사항 |
| --- | --- | --- |
| Δlevel | 0.348 | 0값 19.4% — 성장 정체 유저 다수 |
| Δcombat_power | 2,284,400 | 음수 21.8% — Winsorize(P5-P95) 필요 |
| Δunion_level | 38.43 | 음수 3행 (계정 초기화 의심) → 0 처리 |
| Δarcane_symbol | 0.0 (중앙값) | **0값 65.8%** — 아케인 포화(84.0%)로 인한 점질량 문제 |
| Δauthentic_symbol | 0.87 | 연속 분포, 정상 범위 |

### 가설 외 발견 인사이트

1. **아케인심볼 포화 문제**: 전체 84.0%(1,229명)가 아케인 최대치(120점) 도달 → `avg_monthly_delta_arcane_symbol`의 0값 비율 65.8%. 연속 변수로 사용 부적합 → **`arcane_stagnant` 이진 변수로 대체** (미포화+delta=0인 유저 78명 = 5.3% 의미 있는 신호)

2. **경험치-어센틱심볼 강한 상관**: `log_exp` vs `authentic_symbol_score` 상관 r=0.848 → 두 변수는 중복 정보. 클러스터링에 동시 사용 시 정보 과다 계상 위험

3. **경험치 주차 신호 발견**: `exp_rank_within_level` (같은 레벨 내 경험치 백분위) vs Δlevel의 Spearman r = **−0.113** (p<0.0001) — 레벨 변화 없는 유저일수록 해당 레벨 내 상대 경험치가 높음 → 주차 보조 신호로 활용 가능

4. **다중공선성 없음**: delta 5개 피처 VIF 전원 ≤ 5 (Δlevel: 1.47, Δcp: 1.88, Δunion: 1.66, Δarcane: 1.99, Δauthentic: 3.41) → PCA 불필요

5. **stagnation_score 분포**: 5점(전 피처 정체) 54명(3.7%), 4점 이상 216명(14.8%) — 주차 유저 하한 추정치

### H1/H2/H3 검증 가능성 판정

| 가설 | 판정 | 근거 |
| --- | --- | --- |
| H1 (K-Means/DBSCAN) | ✅ **조건부 가능** | arcane 이진화 + cp Winsorize 후 1,463행 확보 |
| H2 (Chi-Square) | ✅ **가능** | 레벨 4구간 Chi2=663.39, 직업 5계열 Chi2=24.88, 최소 기대 빈도 23.24 (≥5) |
| H3 (Feature Importance) | ✅ **가능** | 주차 프록시 23.7% (클래스 불균형 없음), H1 결과 의존 |

## 🛠️ 전처리 결정 (다음 단계 클러스터링 적용)

EDA 결과를 바탕으로 확정된 전처리 파이프라인:

| 처리 항목 | 방법 | 근거 |
| --- | --- | --- |
| NaN 34행 제거 | listwise deletion | MAR 확인 (레벨 구간별 편향 없음) |
| delta_cp 음수 (21.8%) | Winsorize(P5-P95) | 0 클리핑 시 "성장 없음" 오해 발생 |
| delta_union 음수 3행 | → 0 처리 | 계정 초기화 케이스, 소수 |
| delta_arcane 음수 1행 | → 0 처리 | API 오류 가능성 |
| delta_arcane (65.8% 점질량) | `arcane_stagnant` 이진 변수 대체 | 포화 유저 84% → 연속 변수 부적합 |
| level==300 경험치 | `log_exp` = NaN | 레벨 만렙 시 exp=0 (의미 없음) |

**클러스터링 피처 세트 A (권장)**:

```python
CLUSTER_FEATURES_A = [
    'avg_monthly_delta_level',
    'avg_monthly_delta_combat_power',   # Winsorize 적용
    'avg_monthly_delta_union_level',
    'avg_monthly_delta_authentic_symbol',
    'arcane_stagnant',                  # delta_arcane 이진화 대체
]
```

**클러스터링 피처 세트 B (비교용)**: delta_arcane 원본 유지 후 Silhouette Score 비교
