# 관련 연구 및 차별점

## 1. 본 프로젝트의 위치

본 프로젝트는 Nexon OpenAPI로 직접 관측 가능한 캐릭터 성장 이력과 접속 흔적을 이용해 메이플스토리의 "주차 후보"를 탐색한다. 여기서 주차 후보는 확정 제재 대상이나 봇 계정이 아니라, 270~290레벨의 active/capable 본캐 표본 중 최근 성장 재투자가 낮은 후보군이다.

공개 API에서는 주간 보스 수행, 메소 생산량, 거래, 사냥 좌표, 채팅, 파티 플레이 같은 핵심 행동 로그를 직접 볼 수 없다. 따라서 본 분석은 실제 주차 여부를 단정하지 않고, 운영 검증이나 후속 조사에 넘길 수 있는 후보 라벨과 설명 가능한 rule을 만드는 데 목적이 있다.

## 3. 기존 연구 대비 차별점

1. **대상 문제의 차이**
   - 기존 MMORPG 이상행동 연구는 주로 봇, 골드파밍, RMT, auto-leveling처럼 명시적 악성 행동을 탐지한다.
   - 본 프로젝트는 불법 자동화가 아니라, 고접속·고레벨·보스 가능 조건을 갖춘 캐릭터가 성장 재투자를 멈추거나 낮춘 "주차 후보"를 찾는다.

2. **데이터 접근 조건의 차이**
   - 선행 연구는 회사 내부 서버 로그, 행동 시퀀스, 좌표, 파티, 채팅, 거래, banned account list 등을 사용하는 경우가 많다.
   - 본 프로젝트는 공개 Nexon OpenAPI만 사용한다. 따라서 직접 관측 가능한 전투력, HEXA, 유니온, 심볼, 레벨, 경험치, 접속 스냅샷을 조합해 proxy를 만든다.

3. **라벨 철학의 차이**
   - 기존 봇 탐지 연구는 정상/봇 또는 골드파머/비골드파머 분류 성능을 목표로 하는 경우가 많다.
   - 본 프로젝트는 ground truth가 없음을 전제로 하고, H1의 `is_stagnant_cluster`를 확정 라벨이 아닌 후보 라벨로 둔다. H3 rule 역시 실제 주차 탐지력이 아니라 H1 후보 라벨 근사도로 해석한다.

4. **피처 설계의 차이**
   - 단일 레벨업 속도나 전투력만 보지 않고, 전투력·HEXA·유니온·아케인/어센틱 심볼이라는 서로 다른 성장 시스템의 최근 6개월 delta를 함께 본다.
   - 전투력 감소처럼 부호가 중요한 신호는 signed-log로 보존하고, 접속은 성장 군집화 피처가 아니라 active 표본 통제 조건으로 사용한다.

5. **분석 산출물의 차이**
   - 기존 서비스는 캐릭터 조회, 랭킹, 효율 계산처럼 개인 단위 정보 제공에 강하다.
   - 본 프로젝트는 표본 단위 후보 라벨을 만들고, 레벨 구간/직업 계열 분포 검정(H2)과 설명 가능한 보수 rule(H3)을 통해 왜 해당 후보군이 의미 있는지 분석한다.

## 4. 본 분석의 문제 해결 기여

본 프로젝트의 기여는 "주차 유저를 확정 탐지했다"가 아니라, 공개 API라는 제한된 데이터 조건에서 운영 검토 가능한 후보군을 체계적으로 만드는 데 있다. 내부 로그 기반 연구와 달리 제재 판단을 자동화하지 않고, 공개 관측치로 확인 가능한 성장 정체 패턴만을 사용한다. 기존 메이플 조회 서비스와 달리 개별 캐릭터의 현재 스펙을 보여주는 데서 멈추지 않고, 최근 6개월 변화량과 다중 성장 시스템의 저재투자 패턴을 결합해 후보 라벨, 분포 검정, 설명 가능한 rule까지 연결한다.

따라서 본 분석은 다음과 같이 정리할 수 있다.

- 연구 대상: 봇/골드파머가 아니라 active/capable 주차 후보
- 데이터: 내부 로그가 아니라 공개 Nexon OpenAPI
- 방법: 비지도 후보군 탐색 + DBSCAN 보조 검증 + 분포 검정 + 설명 가능한 rule
- 해석: 확정 판정이 아니라 운영 검증용 후보 라벨
- 차별점: 기존 조회 서비스가 제공하지 않는 장기 성장 정체·저재투자 패턴의 코호트 분석
