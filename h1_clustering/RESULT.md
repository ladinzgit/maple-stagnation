# H1 클러스터링 결과 (6mo 전환 2026-06-05)

## 가설

최근 6개월간 주요 성장 지표가 낮은 활성 캐릭터들은 일반 성장 캐릭터들과 구분되는 성장 정체 군집으로 식별될 것이다.

## 핵심 설계

- **윈도우 = 최근 6개월(2025-12~2026-05)**: 주차는 **현재** 행동(성장 동결)이라 12mo는 과거 활발기가 섞여 정체신호를 희석(recall 0.57). 6mo로 좁히면 recall 0.83 + 고-k 고순도 코어 분리.
- **재현성**: 6mo delta = features_monthly.csv의 **`recent6_delta_*` 컬럼**(collect_features.py 산출). raw 스냅샷 6mo 재산출과 **corr=1.0·median|diff|=0 동등** 검증 → **features_monthly 단독 재현**(monthly_snapshots_raw 불필요). 두 H1 노트북이 12mo `avg_monthly_delta_*`를 `recent6_delta_*`로 override.
- **접속(access) = 통제변인**: 표본 전원 `access_active_months ≥ 10` (100%, min 10 / max 12). 접속이 균일해 군집 축이 못 됨 → access family 클러스터링 제외.
- **주차 후보군 = 클러스터링 결과(`is_stagnant_cluster`) 자체.** 절대-0 동결 게이트 폐기 — 주차 = "상대적 성장 하위 + 활발한 접속".

## 표본

- 본캐 표본: **2,000명** (270~290, 5계열×400, 전원 ≥10/12 접속)
- 클러스터링 표본(df_final): **2,000명** (core delta dropna 제거 0)

## 채택 피처 + 전처리 (4-시스템 다축)

| Alias | 실제 사용 값 | 시스템 |
|---|---|---|
| `cp_slog` | `sign(Δcp)·log1p(\|Δcp\|)` — 전투력 월평균 증가량(winsor p01~p99) signed-log | 전투력 |
| `hexa_avg` | `avg_monthly_delta_hexa` clip≥0 + upper p99 | 헥사 코어 |
| `union_log` | `log1p(clip≥0 avg_monthly_delta_union_level)` | 유니온 |
| `auth_log` | `log1p(clip≥0 avg_monthly_delta_authentic_symbol)` | 어센틱 심볼 |

- **전투력 축은 signed-log 필수**: raw Δcp는 heavy-tail + **음수 19.4%(6mo)**(전투력 감소=주차 신호) → K-Means에서 묻힘. signed-log(부호 보존 + 압축)로 최강 판별축.
- **4-시스템 다축**: 전투력·헥사·유니온·심볼 = 서로 다른 4개 게임 시스템의 보스재화 미재투자를 다면 포착(주차 정의 0.1 정합). 다중공선성 통과(|corr| 최대 0.54 hexa↔auth, VIF<1.6).
- cumEXP·hfrag는 6mo 풀 제외(월별 재산출 비신뢰/raw 없음). 클러스터링 전 `StandardScaler`, 성장 정체 군집 = 스케일 중심좌표 합 최저.

## 선정 (feature_selection.ipynb)

- sil 절대값은 **2피처(0.65) > 3피처(0.53) > 4피처(0.36)** — KMeans 차원 저주 + cp 단일축 지배. 그러나 sil은 cp 단일축 분리를 보상할 뿐 다면 주차 개념을 못 담음.
- **12mo에선 피처 추가가 무익**(recall 0.57 고정·sil만 하락)이었으나, **6mo에선 4피처가 고-k(k=5~6)에서 고순도 코어(162명, enrich 8.6) 분리** → 다축이 판별력 회복.
- 운영정의(미재투자 다면) 정합 위해 **4-시스템 다축 채택(user-pinned)**. 운영점 = elbow k=4(광의).

## silhouette 해석 (주 지표 아님)

silhouette **0.36은 "성장량으로 자연스럽게 잘 갈라진 4개 군집"이라 주장하기엔 낮다.** 그러나 이 표본은 성장량이 **연속 스펙트럼**(전원 이벤트 baseline 성장 + 점진적 둔화)이라 군집 경계가 흐릿한 것이 **정상**이다. H1의 목적도 확정 분류가 아니라 **H2/H3용 주차 후보 라벨 생성**이다. 따라서 H1의 주 평가 지표는 silhouette이 아니라:

1. **정체군 profile** — 전투력 감소 + 4-시스템 저재투자 (위 표).
2. **외부 proxy enrich/recall** — enrich 4.37x, recall 83.0%.
3. **보조 구조 확인** — DBSCAN 교차검증 정합(380↔355). 같은 피처 공간에서 K-Means 정체군집이 밀도 기반 군집과도 상당히 겹치는지 확인한다.
4. **H2/H3 다운스트림 재현성** — 후보 라벨이 분포검정·rule 추출에서 안정적으로 작동하는지.

silhouette은 k 선택의 **보조 신호**로만 쓴다(elbow k=4 채택, best-sil k=6 병기).

## 검증

- 외부 proxy(stag≥4) enrich 4.37×, **recall 83.0%** — 12mo(0.57) 대비 큰 향상. 6mo 최근창이 현재 동결 유저를 더 잘 포착.
- **DBSCAN**: 3군집·noise 2.9%(eps 0.656), 주요 1578/355/8. **K-Means 정체 380 ↔ DBSCAN cluster1(355) 강한 정합**(380 중 355가 동일 dense 군집, 20만 noise) → 같은 피처 공간에서 후보군 구조가 K-Means에만 의존하지 않는다는 보조 근거.

## 해석

"확정 주차 유저"가 아니라 **4-시스템 성장(파워 재투자) 하위 + 고활성** 후보 군집. 공개 API에 보스/메소 기록이 없어 "후보군"이 정확하다. DBSCAN 정합(380↔355)은 **같은 피처 공간에서의 보조적 구조 확인**이며, 실제 주차 여부에 대한 외부 검증은 아니다. 과거 `temporal_external_validation.ipynb`(절대-0 게이트 기반 시간분할 + 현재성 후보 52/39)는 **삭제** — 폐기된 절대-0 정의에 기반하고 6mo 헤드라인과 모순이라 프로젝트에서 제거.
